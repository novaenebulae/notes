Befunge – Langage Bidimensionnel

  1. Inspiration pour la Création :

    • Chris Pressey a créé Befunge pour rendre les programmes impossibles à décompiler, avec une structure si unique qu’il est difficile de comprendre le flux du programme.

  2. Caractéristiques Uniques :

    • Grille Bidimensionnelle : grille bidimensionnelle, permettant à l'exécution de se déplacer non seulement de gauche à droite, mais aussi vers le haut, le bas, et la gauche.
    • Modèle de Pile : Befunge utilise une pile, où les valeurs sont empilées et manipulées.

  3. Limitations :

    • La version originale, Befunge-93, limite le programme à une grille de 80x25 caractères. Cependant, des versions comme Befunge-98 ont étendu cette limite pour permettre des programmes plus longs.

Brainfuck – Langage Minimaliste

  1. Concept Théorique :

    • Machine de Turing : Brainfuck est une implémentation simplifiée d'une machine de Turing.

  2. Utilité et Popularité :

    • Bien qu'il soit peu utilisé pour des projets pratiques, Brainfuck est populaire pour des concours et des défis, où l’objectif est de réaliser des programmes fonctionnels avec un minimum de commandes.

  3. Problème de Lisibilité :

    • Écrire et comprendre le code en Brainfuck est extrêmement difficile, car les commandes sont limitées et en caractères uniques. Certains experts s'amusent à écrire des programmes avancés, tels que des calculateurs et générateurs de Fibonacci.

Whitespace – Programmation Invisible

  1. Concept d’Invisibilité :

    • Whitespace utilise des caractères invisibles (espaces, tabulations et retours à la ligne) pour coder, ce qui fait que le code n'est visible qu’en mode éditeur spécialisé. Cela permet même de cacher un programme Whitespace dans le code source d'un autre langage sans interférer avec celui-ci.

  2. Application Humoristique :

    • Whitespace a été créé principalement comme une blague, soulignant les aspects absurdes de certains langages en poussant la syntaxe invisible à l'extrême.

  3. Structure en Trois Commandes :

    • Espaces : utilisés pour les opérations sur la pile.
    • Tabulations : pour les opérations de contrôle de flux et arithmétiques.
    • Retour à la ligne : pour marquer la fin d'instructions ou du programme. Cette séparation pousse le concept minimaliste tout en offrant une certaine complexité de logique.
